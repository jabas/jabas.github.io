Form Examples

Form examples demonstrate a few techniques for form layout and accessibility.

Accessible Forms

▼

Basic Accessible Form

  • Include matching for and id values unique to each label and input pair to associate the two.
  • Avoid using the float label pattern.
  • Avoid using placeholder text as it makes the input look complete.
Your Full Name
What option(s) do you prefer?
  • Only use a fieldset with legend to group associated inputs (e.g. multiple address inputs or credit card month/year inputs) or a set of radios or checkboxes. Some assistive tools will read the contents of the legend before each nested input to maintain context for a user.
  • The legend can be visually hidden with the .sr-only utility class.
  • Stack checkbox and radio buttons when possible.

Helper or instructional text for the input

  • When an input requires additional help or descritive text match aria-describedby and id values.
  • Indicate that an input is required by adding the aria-required="true" attribute. By default this is false and does not need set unless it will be true.
  • In most cases you do not need to label an input as required considering a majority of our inputs are. Instead, mark optional inputs in the label text, e.g., “Label (optional).”
Helper or instructional text for the input
  • An invalid input should include the aria-invalid="true" attribute.
